GPGMK

A GPU powered, 3D geometric modeling kernel

View project on GitHub

Introduction

Geometric modeling kernels play an important role in modern CAD systems. They are the core of modern CAD systems, providing geometric modeling algorithms, and base classes to store and implement constructive solid geometry objects. Without them, modern CAD applications most likely would not exist in their current state.

However, current geometric modeling kernels are heavily CPU based, and currently are not optimized for parallel processing even on CPUs. The algorithms implemented in these systems are commonly implemented in parallel for performance benefits, but lack support for GPUs, the parallel nature of which would give large performance boosts to these algorithms.

As well, current CAD systems are for the most part platform dependent, usually depending on Windows. This dependency on closed source software makes it hard to improve the current state of the arts performance because of this lack of accessibility to the geometric algorithms source code.

gpgmk is, to the best of my knowledge, the first geometric modeling kernel to offer GPU support for CAD algorithms.

Project Goals

The goal of this project is to implement a set of standard CAD algorithms, including filleting/rounding, chamfering, extrude, rotate, loft, sweep, union, difference, and, or, and xor. These algorithms should be implemented with as much GPU support as possible, in order to gain the most performance from GPU utilization as possible.

Visualization functionality should also be provided, so that the results of these operations can be visually verified and checked against other CAD systems.

As well, these functions should be implemented in as cross platform manner, using cross platform technologies, so that the largest number of people can take advantage of the generated product.

In short, the major goals of this project can be summarized as follows:

  • Implement a standard set of CAD algorithms.
  • Add GPU support wherever feasible to increase performance.
  • Use cross platform technologies wherever possible, to increase the portability of geometric modeling kernels.
  • Provide visualization support, so results can be visually verified.
